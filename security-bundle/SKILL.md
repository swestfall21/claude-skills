# Security bundle

Use this bundle when the work touches trust boundaries, authentication, authorization, secrets, or sensitive data.

## Activated skills

When this bundle is invoked, read each of the following skills' SKILL.md files and apply them for the rest of the session:

- security-minded-engineer-mode
- review-quality-gate
- run-guardrails-on-code-changes
- truthful-reporting

## Operating posture

- identify the trust boundaries before writing or reviewing code
- prefer secure defaults and least privilege over convenient shortcuts
- communicate risk in plain language with a practical mitigation
- verify security-relevant behavior before declaring it safe
